# システム構成の概要

この設定は、Raspberry PiからPCにビデオをストリーミングし、PCからRaspberry Piに制御コマンドを送信することを可能にする、有線ネットワーク接続を前提としています。

## 1. Raspberry Pi サーバー (`only_rasp1/pi_server.py`)

*   **役割:** サーバーとして機能し、カメラからビデオをキャプチャし、制御コマンドを受信します。
*   **ビデオストリーミング (UDP - ポート 5005):**
    *   `cv2.VideoCapture(0)` を使用してRaspberry Piのカメラにアクセスします。
    *   ビデオフレームをキャプチャし、JPEG形式 (品質50%) に圧縮し、設定されたPC_IP (例: "192.168.50.10") の `VIDEO_PORT` (5005) にUDPパケットとして送信します。
    *   軽量化のため、フレームは320x240にリサイズされます。
    *   約30 FPSにフレームレートを制限するために `time.sleep(0.03)` が含まれています。
    *   ビデオ送信は別のスレッドで実行されます。
*   **制御コマンド受信 (TCP - ポート 5006):**
    *   `MY_IP` ("0.0.0.0", すべてのインターフェース) および `CONTROL_PORT` (5006) で着信TCP接続をリッスンします。
    *   接続後、連続してデータを受信します。
    *   受信したデータをUTF-8でデコードし、改行で分割して1つのパケット内の複数のコマンドを処理します。
    *   "FORWARD"、"BACK"、"LEFT"、"RIGHT"、"STOP" などのコマンドを処理します。
    *   **主な機能:** 受信したコマンドが `last_command` と異なる場合にのみコマンドを処理および表示し、状態ベースの制御 (例: 現在プレースホルダーであるGPIO操作) を最適化します。
    *   制御受信はメインスレッドで実行されます。

## 2. PC クライアント (`RaspberryPi/pc_client.py`)

*   **役割:** クライアントとして機能し、Raspberry Piから受信したビデオを表示し、制御コマンドを送信します。
*   **ビデオ受信と表示 (UDP - ポート 5005):**
    *   `MY_IP` ("0.0.0.0") および `VIDEO_PORT` (5005) でUDPパケットをリッスンします。
    *   圧縮されたJPEGデータを受信し、`cv2.imdecode` を使用してビデオフレームにデコードし、「Raspberry Pi Camera (Low Latency)」というウィンドウに表示します。
    *   ビデオ受信は別のスレッドで実行されます。
    *   'q' を押すとビデオウィンドウが閉じます。
*   **制御コマンド送信 (TCP - ポート 5006):**
    *   `RPI_IP` (例: "192.168.50.20") の `CONTROL_PORT` (5006) にTCP接続を確立しようとします。
    *   `pygame` を使用してキーボード入力 (WASDキー) を検出します。
    *   キー入力に基づいて:
        *   'W' は "FORWARD" を送信します。
        *   'S' は "BACK" を送信します。
        *   'A' は "LEFT" を送信します。
        *   'D' は "RIGHT" を送信します。
        *   移動キーが押されていない場合、"STOP" が送信されます。
    *   コマンド (改行文字が付加されたもの) をUTF-8でエンコードして送信します。
    *   コマンドの頻繁な送信を防ぐために `time.sleep(0.05)` が含まれています。
    *   制御送信はメインスレッドで実行されます。

## 相互作用フロー:

1.  Raspberry Pi (`pi_server.py`) が起動し、ビデオキャプチャを開始し、UDPを介してPCにストリーミングを開始します。同時に、TCPサーバーを設定して制御コマンドをリッスンします。
2.  PC (`pc_client.py`) が起動し、Raspberry PiのTCP制御ポートに接続を試み、同時にRaspberry PiからのUDPビデオストリームをリッスンし始めます。
3.  TCP接続が確立されると、PCクライアントはWASDキー入力を継続的に監視します。
4.  キー入力に基づいて、PCクライアントは対応する制御コマンド ("FORWARD"、"BACK"、"LEFT"、"RIGHT"、"STOP") をTCP経由でRaspberry Piに送信します。
5.  Raspberry Piサーバーはこれらのコマンドを受信し、処理し (新しいコマンドの場合のみ)、理想的にはGPIOアクションをトリガーします (現在はプレースホルダーです)。
6.  PCクライアントは、Raspberry Piからの着信ビデオストリームを表示します。
